# WorldFish｜创世鱼缸

> 面向文艺创作者的 AI 世界观构建、管理与动态推演工具。

WorldFish（创世鱼缸）是一个帮助创作者构建、整理、维护和推演世界观的 AI 产品。

它适用于小说作者、动画编剧、游戏策划、OC 爱好者、TRPG 创作者、Wiki 维护者以及虚构作品社区工作者。用户可以从零开始创建世界观，也可以导入已有文本，让 AI 自动提取其中的人物、地点、势力、事件、时间线与设定信息，并进一步通过 Agent 对话进行修改、补全和推演。

WorldFish 不是一个通用聊天机器人，也不是单纯的 AI 小说生成器。它的核心目标是成为一个可长期维护的世界观管理系统，让创作者能够把零散灵感、文本资料和设定内容沉淀为结构化的创作资产。

---

## 核心定位

WorldFish 的定位是：

> 一个面向 AI 时代文艺创作者的世界观基础设施。

它关注的问题包括：

- 如何从零搭建一个完整世界观？
- 如何把已有小说、剧本或设定文本整理成结构化资料？
- 如何管理复杂的人物、地点、组织、势力、事件和时间线？
- 如何让 AI Agent 参与世界观构建与修改？
- 如何基于已有世界观进行未来剧情、历史变化和 if 线推演？
- 如何用地图系统管理世界中的空间关系？

## 已支持功能

### 1. 世界观构建系统

WorldFish 提供结构化的世界观管理能力，用于记录和维护世界中的核心信息。

支持的内容包括：

- 世界基础设定
- 实体
- 地点
- 势力
- 组织
- 角色
- 事件
- 时间线
- 设定说明
- 世界规则
- 关联关系

用户可以逐步搭建自己的世界观，而不是只依赖零散文档或聊天记录。

---

### 2. AI Agent 世界观助手

WorldFish 支持通过 Agent 与世界观数据进行对话。

用户可以用自然语言要求 Agent：

- 创建世界观设定
- 修改已有设定
- 补全角色、地点、势力和事件
- 检查设定一致性
- 查询世界观信息
- 协助整理已有文本
- 生成新的世界观内容
- 辅助进行世界观推演

Agent 的作用不是简单聊天，而是作为世界观构建与维护过程中的 AI 协作者。

---

### 3. AI 世界观提取

用户可以输入已有文本，让 AI 从中提取世界观信息。

适用文本包括：

- 小说原文
- 剧本片段
- 游戏设定文档
- 角色小传
- 世界观介绍
- Wiki 文本
- 社区资料
- 作品设定集

系统会尝试从文本中识别并整理：

- 人物
- 地点
- 组织
- 势力
- 事件
- 时间线
- 世界规则
- 角色关系
- 重要设定

这可以帮助创作者和社区维护者减少大量手动整理工作。

---

### 4. RAG 知识库

WorldFish 支持基于用户输入内容构建 RAG 知识库。

这使得 Agent 在回答问题、修改设定或进行推演时，可以参考用户已有的世界观资料，而不是完全依赖模型自身记忆。

RAG 知识库可用于：

- 查询已有世界观信息
- 降低设定遗漏
- 提高回答一致性
- 支持长篇作品资料管理
- 支持已有文本分析
- 为推演提供上下文依据

---

### 5. 世界观推演

WorldFish 支持基于已有世界观进行推演。

用户可以指定推演目标，例如：

- 推演未来剧情发展
- 推演某个国家的兴衰
- 推演一场战争的后续影响
- 推演角色命运变化
- 推演某个事件改变后的 if 线
- 推演不同势力之间的关系变化
- 推演世界在特定时间后的发展

推演功能的目标不是替代创作者做最终决定，而是提供可能的发展路径，帮助创作者发现新的剧情方向、设定矛盾和故事冲突。

---

### 6. 结构化地图系统

WorldFish 已支持世界观地图系统。

地图系统用于管理世界中的空间结构，并为世界观推演提供地理基础。

当前地图系统以结构化地图单元为基础，可以用于记录和管理：

- 地形
- 区域
- 国家
- 势力范围
- 城市
- 资源
- 事件发生地
- 战争区域
- 角色活动地点
- 世界观实体与地图位置的关联

地图系统不是单纯的图片展示，而是世界观数据库的一部分。

它可以帮助用户回答：

- 某个国家控制哪些区域？
- 某个城市位于哪里？
- 某种资源分布在哪些地方？
- 某个事件发生在哪片区域？
- 某个角色的活动范围在哪里？
- 某场战争影响了哪些地区？
- 世界中的空间关系如何变化？

地图功能为后续的战争推演、势力扩张、事件扩散、资源争夺、文化传播和 if 线地图变化提供了基础。

## 典型使用场景

### 场景一：从零创建世界观

用户可以输入一个模糊想法：

> 我想创建一个蒸汽朋克风格的大陆，有三个国家，其中一个国家掌握能源技术，另一个国家信奉古老宗教。

WorldFish 可以协助用户逐步生成：

- 世界基础设定
- 地理结构
- 国家与势力
- 历史事件
- 角色关系
- 关键冲突
- 时间线
- 地图区域
- 后续发展方向

用户可以继续通过 Agent 对话不断修改和补全世界观。

---

### 场景二：整理已有小说或剧本

用户可以导入已有文本。

WorldFish 会从文本中提取人物、地点、事件、势力、时间线和设定规则，并将其整理为结构化世界观资料。

之后用户可以：

- 查询已有设定
- 检查设定矛盾
- 补全缺失信息
- 构建知识库
- 生成地图关联
- 推演未来剧情

---

### 场景三：构建作品 Wiki

对于 Wiki 维护者或社区资料整理者，WorldFish 可以帮助从原文或资料中提取结构化信息。

例如：

- 角色条目
- 地点条目
- 势力条目
- 时间线条目
- 事件条目
- 设定条目
- 地图区域信息

这可以降低资料整理和维护成本。

---

### 场景四：世界观地图管理

用户可以为自己的世界创建地图，并将世界观信息与地图区域绑定。

例如：

- 给某片区域设置为山地
- 给某个国家标记控制范围
- 给某个区域添加铁矿资源
- 把一场战争绑定到多个地图区域
- 把角色出生地绑定到地图单元
- 查看某个势力在地图上的分布

地图系统让世界观不再只是文本，而是具备明确的空间结构。

---

### 场景五：if 线与未来推演

用户可以基于当前世界观提出推演问题：

> 如果主角没有加入反抗军，这个帝国会怎样发展？

或者：

> 从第三纪元 420 年开始，推演未来 30 年的国际局势。

WorldFish 会基于已有设定、知识库、事件、时间线和地图信息，生成可能的发展路径。

---

## 当前阶段

WorldFish 当前处于 MVP 阶段。

当前重点是验证以下核心能力：

- 世界观是否能被结构化管理
- AI 是否能有效提取已有文本中的设定信息
- Agent 是否能辅助用户构建和修改世界观
- RAG 是否能提高世界观查询和回答的一致性
- 推演功能是否能为创作者提供有效参考
- 地图系统是否能帮助用户管理空间关系

MVP 阶段不会追求功能大而全，而是优先保证核心创作链路可用。
