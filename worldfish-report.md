## 核心摘要

:::callout{type="info", icon="🐟", title="品牌核心定位"}
WorldFish（创世鱼缸）是我正在开发的面向创作者、开发者、叙事团队与互动娱乐用户的 **AI 驱动型世界观构建与推演平台**。
:::

我的长期愿景不是做单一的 TRPG 工具，也不是普通 AI 写作助手，而是构建一个围绕「**创造世界、理解世界、推演世界、进入世界**」的完整创作与互动生态。

在我规划的成熟形态中，WorldFish 将由 **三大核心能力** 构成：

:::features
@feature{icon="🌍", title="AI 驱动的世界观构建平台", description="用户可以通过自然语言、设定文档、小说正文、游戏策划案、剧本、PDF、Markdown 等资料输入，由 AI 自动提取人物、地点、组织、势力、事件、规则体系等世界观元素，并生成结构化、可编辑、可检索、可协作的世界观数据库。"}
@feature{icon="🔮", title="世界推演引擎", description="用户可以基于既有世界观提出推演问题，系统将根据世界观设定、实体关系、时间线、冲突压力、资源分布与叙事风格进行多轮推演，生成因果链、事件分支、实体状态变化、时间线更新和剧情指导建议。"}
@feature{icon="🎭", title="互动叙事与 TRPG 跑团平台", description="在世界观与推演能力之上，承载 TRPG 跑团、多人互动叙事、AI Dungeon Master、角色扮演、虚拟桌面、剧情分支探索、用户共创世界等场景，使创作者不仅能「写出世界」，还能「运行世界」和「进入世界」。"}
:::

:::card{title="总体判断"}
WorldFish（创世鱼缸）具备成为**中文创作者世界观基础设施**的潜力。其长期竞争力不在于单点 AI 生成，而在于构建一个围绕「世界」的创作、推演和互动平台。
:::

从我对市场格局的观察来看，全球和国内都缺少一款真正打通「**世界观构建 — 世界推演 — 互动叙事/跑团**」的一体化平台。海外产品如 World Anvil、LoreKeeper、Friends & Fables、Foundry VTT 分别在结构化世界观管理、AI 世界观辅助、AI 跑团、虚拟桌面生态方面形成优势，但往往只覆盖链路中的一部分；国内产品则更多聚焦 AI 剧情生成、轻量化跑团或泛娱乐文字互动，缺少面向专业创作者与内容团队的世界观资产平台。

## 1. 项目背景与研究方法论

### 1.1 研究目标

本报告旨在为 WorldFish（创世鱼缸）的品牌化运营、产品定位与市场发展策略提供支撑，重点回答以下问题：

- WorldFish 未来成熟形态应如何定位；
- 它与现有世界观工具、AI 写作工具、TRPG 平台和互动叙事产品有何区别；
- 国内外同类产品的能力边界与市场短板在哪里；
- WorldFish 在国内市场有哪些差异化机会；
- 「MIT 开源单人核心 + 闭源云平台」的商业模式是否可行；
- 品牌化运营阶段应优先强调哪些卖点。

### 1.2 核心定义

:::quote{author="WorldFish 品牌定义"}
**WorldFish（创世鱼缸）是一个 AI 驱动的世界观构建、世界推演与互动叙事平台。它帮助用户创造一个世界、整理一个世界、推演一个世界，并最终让用户进入这个世界。**
:::

这里的「世界观」不只指 TRPG 模组，而是泛指所有需要设定体系和叙事逻辑支撑的内容资产，包括：

:::grid
- 游戏世界观
- 小说世界观
- 影视动画设定
- 漫画与 IP 宇宙
- TRPG 跑团世界
- 虚拟角色与虚拟偶像背景
- 品牌故事宇宙
- 历史架空模拟
- 教育与社会沙盘
- AI Agent 所处的虚拟环境
- 用户共创的多人互动世界
:::

### 1.3 WorldFish 的市场边界

我规划的成熟产品形态可拆分为 **四层架构**：

:::steps
@step{number="1", title="世界观资产层", description="管理人物、地点、组织、势力、事件、规则、文化、历史、历法、地图等基础资料，解决资料分散、关系混乱、设定不一致等问题。"}
@step{number="2", title="AI 构建层", description="通过 AI 从用户输入的自然语言、文档、小说、剧本、策划案等内容中提取结构化世界观，降低从零搭建的高成本。"}
@step{number="3", title="世界推演层", description="基于世界观数据进行多轮推演，回答「如果……会怎样」的问题，生成剧情分支、历史演化、势力变化和世界状态更新。"}
@step{number="4", title="互动运行层", description="支持多人协作、联机跑团、互动叙事、AI DM、AI NPC、角色扮演、虚拟桌面、场景生成与世界共创。"}
:::

## 2. WorldFish 产品愿景与能力体系

### 2.1 品牌定位

:::callout{type="info", icon="🎯", title="品牌定位语"}
**WorldFish 创世鱼缸：让你的世界自己生长。**
:::

我认为 WorldFish 的品牌中文名「创世鱼缸」具有很强的隐喻价值：

- 「**创世**」代表世界的生成、创造和设定；
- 「**鱼缸**」代表一个可观察、可干预、可演化的封闭世界系统；
- 用户既是 **造物主**，也是 **观察者**、**导演**、**主持人** 和 **参与者**。

我推荐的品牌定位语包括：

| 场景 | 定位语 |
|---|---|
| 通用版 | WorldFish 是一个 **AI 驱动的世界观构建与推演平台**。 |
| 创作者版 | 把你的设定文档变成一个 **可推演、可协作、可运行** 的世界。 |
| 游戏版 | 为游戏世界观、阵营关系和剧情分支提供 **AI 推演引擎**。 |
| 小说版 | 帮长篇作者管理 **人物、势力、时间线和设定一致性**。 |
| TRPG 版 | 让主持人用 AI 构建世界、推演剧情，并带玩家 **进入其中**。 |
| 品牌版 | **创造一个世界，然后让它自己生长。** |

### 2.3 三大核心能力详述

:::columns{cols="3"}
:::card{title="世界观构建平台"}
传统世界观创作依赖文档、表格等通用工具。这些工具可以记录内容，但无法真正理解世界观结构，也无法自动推演设定变化。

WorldFish 将创作者输入转化为 **结构化世界观数据库**，每一个人物、地点、组织、事件都成为可管理、可检索、可推演的对象。
:::

:::card{title="世界推演引擎"}
将静态设定变成动态系统。用户不只是问「这个角色是谁」，而是可以问：

- 如果这个角色死亡，世界会怎样变化？
- 如果魔法体系被工业化，社会阶层如何重组？
- 如果时间推进 100 年，这个文明会演化成什么样？

推演结果可以指导游戏剧情分支、RPG 任务线、小说章节走向等。
:::

:::card{title="TRPG 与互动叙事平台"}
WorldFish 将前两层能力自然延伸到跑团场景，与传统 VTT 的区别在于：**不只是把桌面搬到线上，而是让整个世界成为可运行的叙事系统。**

玩家行为产生新事件 → 新事件反向写回世界状态 → 世界因玩家行为而持续演化。
:::
:::

### 2.4 面向多行业的应用场景

我规划 WorldFish 的服务对象不应局限于 TRPG，而应覆盖泛创作与互动娱乐行业。

:::grid{min_width="280", gap="20"}
:::card{title="🎮 游戏开发"}
开放世界设定、RPG 阵营关系、NPC 背景与任务线、世界事件推演、多结局剧情模拟、游戏策划文档导出。
:::

:::card{title="📖 小说与网文"}
长篇人物关系维护、宗门/国家/势力体系管理、时间线与伏笔整理、剧情分支推演、设定一致性检查。
:::

:::card{title="🎬 影视动画与漫画"}
IP 宇宙设定管理、角色关系图谱、季度剧情推演、世界观 Bible 生成、衍生作品设定统一。
:::

:::card{title="🎲 TRPG 与互动叙事"}
模组准备、AI DM、NPC 对话、即兴剧情应对、玩家行为后果推演、跑团记录回写。
:::

:::card{title="🤖 虚拟 IP 与 AI 角色"}
虚拟角色背景世界、AI 角色长期记忆、角色关系与事件历史、粉丝共创设定、多角色互动剧场。
:::

:::card{title="🏛️ 品牌叙事与文旅"}
品牌故事宇宙、城市/景区文化世界观、沉浸式剧本、互动营销活动、文化 IP 延展。
:::

:::card{title="📚 教育与模拟"}
历史事件推演、社会结构模拟、政治/经济/文化系统演化、课堂互动沙盘、多 Agent 场景实验。
:::

## 3. 海外同类市场竞品分析

:::callout{type="info", icon="🌐", title="海外市场格局"}
海外市场的成熟度显著高于国内市场，已形成「结构化世界观工具、VTT 跑团平台、AI 叙事工具」三类产品并行发展的格局。但在我观察中，**真正将「世界观构建 + AI 推演 + 互动运行」完整打通的平台仍然稀缺。**
:::

### 3.1 竞品矩阵

:::comparison
@comparison-row{pro="🌍 World Anvil\n结构化知识库能力强、模板成熟、社区生态完善", con="AI 能力辅助性、推演能力不足、中文支持弱、学习曲线高"}
@comparison-row{pro="🧠 LoreKeeper\nAI 世界观生成能力强、TRPG 结合紧密、代表 AI 原生方向", con="中文本地化不足、国内题材适配有限、跨行业边界待拓展"}
@comparison-row{pro="🎲 Friends & Fables\nAI DM 体验突出、跑团娱乐属性强、轻量用户友好", con="非专业世界观资产平台、专业工作流支撑不足、结构化深度有限"}
@comparison-row{pro="🖥️ Foundry VTT\nVTT 功能强大、插件生态成熟、自托管心智明确", con="非世界观构建平台、AI 依赖插件、非技术用户门槛高"}
@comparison-row{pro="🗺️ LegendKeeper\n轻量可视化世界观管理、地图式交互", con="功能深度有限、AI 能力不足"}
:::

### 3.2 竞品启示总结

:::highlights{title="对各竞品的战略启示"}
- **World Anvil**：借鉴结构化管理和模板体系，但在 AI 生成、世界推演、推演回写和中文语境上形成差异化。
- **LoreKeeper**：重要参照对象，但不应只复制 TRPG 定位，应扩展到游戏、小说、影视动画等更广泛场景。
- **Friends & Fables**：借鉴 AI DM 和互动体验，但核心差异放在「世界观资产 + 推演引擎 + 多场景内容生产」。
- **Foundry VTT**：更像未来可对接的**下游运行平台**，WorldFish 的差异化在于上游世界观生成、推演和叙事指导。
:::

### 3.3 其他海外产品速览

| 产品 | 定位 | 对 WorldFish 的参考意义 |
|---|---|---|
| **LegendKeeper** | 轻量可视化世界观管理 | 地图式交互体验 |
| **WorldSmith** | AI 快速生成 TRPG 内容 | 轻量生成入口 |
| **NovelAI** | 高质量叙事生成 | 文风模仿和长篇文本体验 |
| **Campfire** | 写作者世界观和小说规划 | 作者工作流 |
| **Kanka** | RPG campaign 管理 | 开源与社区路线 |

:::

| 竞品 | 与 WorldFish 的关系 |

## 6. WorldFish 的核心竞争力

:::kpi-row

@kpi-card{value="因果推演", label="从「剧情生成」到「因果推演」", delta="生成→推演", delta_trend="up"}
@kpi-card{value="可运行世界", label="从「跑团工具」到「可运行世界」", delta="桌面→宇宙", delta_trend="up"}
@kpi-card{value="开源生态", label="从「闭源工具」到「开源核心 + 云生态」", delta="闭源→混合", delta_trend="up"}
:::

### 6.1 五大核心竞争力详解

:::grid{min_width="300", gap="20"}
:::card{title="1. 从「AI 写作」升级为「AI 创世」"}
普通 AI 写作工具生成的是文本，而 WorldFish 生成和维护的是**世界**。文本是一次性的，世界是可持续运行的。
:::

:::card{title="2. 从「设定文档」升级为「世界观资产」"}
传统设定文档难以被检索、推演和复用。WorldFish 将人物、事件、地点、势力、规则、地图等拆解为**资产对象**，使它们可以被 AI 调用、被团队协作、被推演引擎更新、被跑团平台运行。
:::

:::card{title="3. 从「剧情生成」升级为「因果推演」"}
普通 AI 剧情生成只关注文本流畅度，而 WorldFish 的推演引擎强调：**世界状态、事件因果、实体变化、时间线连续、设定一致性、规则约束、后果回写**。
:::

:::card{title="4. 从「跑团工具」升级为「可运行世界」"}
传统 VTT 是桌面工具，WorldFish 的长期目标是让**整个世界观可运行**。跑团只是其中一个入口，未来可扩展到 AI 互动剧、多人共创世界、虚拟 IP 剧场等场景。
:::

:::card{title="5. 从「闭源工具」升级为「开源核心 + 云生态」"}
MIT 开源单人核心吸引早期开发者和创作者，降低信任门槛；闭源云平台承载高价值商业功能。这种混合模式有利于在早期建立口碑，在中后期形成可持续收入。
:::
:::

## 7. 中国市场综合潜力分析

### 7.1 五大市场机会

:::highlights{title="中国市场五大机会"}
:::

:::grid{min_width="280", gap="20"}
:::card{title="📦 世界观基础设施缺失"}
国内游戏、网文、短剧、动画、漫画、TRPG、虚拟 IP 等内容行业都依赖世界观，但多数团队仍使用通用文档工具管理设定。痛点包括：设定分散、时间线混乱、人物关系复杂、团队协作困难。WorldFish 可以成为创作团队的**世界观资产中台**。

:::card{title="📖 中文网文天然入口"}
中文网文、玄幻、仙侠、科幻、都市异能等内容天然具有复杂世界观：大量人物、多层势力、等级体系、地图区域、技能体系、长时间线。非常适合 WorldFish 做 **AI 提取、图谱生成、时间线整理和剧情推演**。

:::card{title="🎮 游戏行业推演需求"}
开放世界、RPG、SLG、模拟经营、剧情向游戏都需要大量世界观设计。WorldFish 可帮助游戏团队生成世界框架、管理阵营与 NPC、推导任务线、模拟势力变化、检查剧情逻辑。
:::

:::card{title="🎲 TRPG 高粘性种子用户"}
TRPG 用户天然需要世界观、NPC、地图、事件、玩家行为后果、即兴剧情推演、长期战役记录。他们能快速理解「让世界自己生长」的价值，是品牌早期的**重要社区入口**。
:::

:::card{title="🤖 虚拟 IP 与 AI 角色增量"}
随着 AI 角色、虚拟陪伴、虚拟偶像、AI Agent 发展，角色需要稳定背景、记忆、关系和世界环境。WorldFish 可为 AI 角色提供「**世界观底座**」，让角色生活在持续演化的世界中。
:::

## 11. 风险提示

:::grid{min_width="300", gap="20"}
:::alert{type="warning", title="⚠️ 定位过窄风险"}
如果 WorldFish 只被市场理解为 TRPG 工具，会限制其想象空间。品牌化运营必须强调「**世界观构建与推演平台**」，TRPG 是重要场景之一，而不是全部。
:::

:::alert{type="warning", title="⚠️ 定位过泛风险"}
如果同时讲游戏、小说、影视、TRPG、教育、品牌、AI Agent，可能导致早期用户不知道产品到底解决什么问题。建议早期传播统一在「**AI 世界观构建与推演**」这一核心。
:::

:::alert{type="error", title="⚠️ AI 质量风险"}
世界推演对一致性、因果和长期记忆要求高。如果 AI 生成内容无法遵守设定，用户会失去信任。需要持续强化**结构化上下文、状态账本、一致性校验和可解释推演**。
:::

:::alert{type="error", title="⚠️ 成本风险"}
长文本提取、RAG、推演和 AI DM 都可能产生较高模型成本。云平台需要建立清晰的**额度、缓存、增量解析和成本控制策略**。
:::

:::alert{type="warning", title="⚠️ 开源与闭源边界风险"}
MIT 开源核心需要与闭源云平台形成合理边界。应避免开源版本过弱导致社区不认可，也避免云平台缺少付费价值。
:::

:::alert{type="warning", title="⚠️ 合规风险"}
若服务国内用户并提供云端平台，需要关注：ICP 备案、公安备案、用户数据存储、生成式 AI 内容管理、内容审核、未成年人保护、用户上传内容版权、隐私政策和用户协议。
:::
:::
